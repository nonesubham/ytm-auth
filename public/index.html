Authentication Service

Privacy Notice

We do not store any user details, credentials, or tokens. This service only facilitates the OAuth flow and immediately displays your token, which you need to pass to ytm-term. No personal data is retained on our servers.

About This Service

This website is part of the open-source project ytm-term, a terminal-based YouTube Music player.

This service securely obtains an access token after you log in with Google. The ytm-term application uses this token to access your YouTube Music playlists.

This step is optional and only required if you want to play your private playlists in the terminal application.

How It Works

  • Click the "Authenticate with Google" button below
  • Log in to your Google account and grant the necessary permissions
  • You will receive an access token
  • ytm-term uses this token to access your YouTube Music data
# After authentication, use the token in ytm-term:
$ ytm-term --token <your_token_here>

Privacy & Security

This is an open-source project. We host this service to protect our OAuth credentials from being misused.

You can verify our source code and privacy practices:

  • ytm-term: github.com/nonesubham/ytm-term
  • This auth service: github.com/nonesubham/ytm-auth

Alternative Options

If you prefer not to use this hosted service:

  • Self-host: Clone and host your own version (instructions in documentation)
  • Manual setup: Set up your own OAuth 2.0 credentials, download the JSON file, and pass it to ytm-term
# Manual OAuth setup example:
$ ytm-term --oauth-file /path/to/your/credentials.json
